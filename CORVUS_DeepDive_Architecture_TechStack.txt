Deep Dive Arsitektur dan Tech Stack

Project: CORVUS – AI Reading Workspace

CORVUS merupakan sebuah Chrome Extension berbasis Artificial
Intelligence yang berfungsi sebagai asisten membaca. Tujuan utama sistem
bukan hanya menghasilkan ringkasan artikel, tetapi memahami konteks
halaman web, menyimpan pengetahuan pengguna, serta menyediakan berbagai
fitur interaktif seperti tanya jawab, keyword extraction, flashcard,
quiz, mind map, dan knowledge graph. Seluruh kemampuan AI menggunakan
API sehingga model dapat diganti dengan mudah tanpa mengubah arsitektur
utama aplikasi.

Arsitektur Sistem

Arsitektur dibagi menjadi beberapa lapisan agar mudah dikembangkan,
dipelihara, dan diskalakan.

1. Presentation Layer (Frontend)

Chrome Extension menjadi antarmuka utama pengguna. Content Script
membaca halaman aktif, kemudian Side Panel menampilkan seluruh hasil
analisis AI. Popup digunakan untuk pengaturan, login, dan shortcut.

Komponen: - Chrome Extension (Manifest V3) - React - TypeScript -
Tailwind CSS - Chrome Side Panel API - Chrome Storage API

Tanggung jawab: - Mengambil URL halaman. - Mengambil konten artikel. -
Menampilkan hasil AI. - Mengirim permintaan ke backend. - Menyimpan
preferensi lokal.

2. Content Extraction Layer

Layer ini bertugas mengambil isi utama halaman dan membuang elemen yang
tidak diperlukan.

Pipeline: 1. Mengambil DOM halaman. 2. Menghapus iklan, navbar, footer,
sidebar, komentar. 3. Mengekstrak artikel utama menggunakan Mozilla
Readability. 4. Membersihkan whitespace dan HTML. 5. Mengirim teks
bersih ke backend.

Teknologi: - Mozilla Readability - DOM Parser

3. API Gateway / Backend

Backend menjadi pusat orkestrasi seluruh layanan AI.

Tanggung jawab: - Autentikasi pengguna. - Validasi request. - Rate
limiting. - Logging. - Caching. - Routing ke layanan AI. - Penyimpanan
histori.

Teknologi: - FastAPI - Uvicorn - Pydantic - JWT Authentication - Redis
Cache

4. AI Processing Layer

Seluruh fitur AI menggunakan API LLM.

Alur kerja: Request → Prompt Builder → LLM API → Structured JSON →
Backend → Extension.

Fitur AI: - Ringkasan (short, medium, detailed). - Keyword Extraction. -
Insight Generation. - Question Answering. - Flashcard Generator. - Quiz
Generator. - Explain Like I’m Beginner. - Vocabulary Explanation. -
Reading Difficulty. - Sentiment Analysis. - Action Items. - Citation
Suggestion.

Prompt dikembalikan dalam format JSON agar frontend mudah merender
hasil.

5. Embedding & Knowledge Layer

Setiap artikel yang dianalisis akan diubah menjadi embedding sehingga
dapat dicari kembali menggunakan semantic search.

Fungsi: - Riwayat pembelajaran. - Chat dengan artikel lama. - Similar
article recommendation. - Knowledge Graph. - Personal memory.

Teknologi: - Embedding API - Qdrant atau ChromaDB

6. Database Layer

Database menyimpan data pengguna dan histori.

Data yang disimpan: - User. - Bookmark. - Highlight. - Notes. - Reading
History. - AI Result. - Preferences.

Teknologi: - PostgreSQL - SQLAlchemy

7. Analytics Layer

Menghasilkan statistik belajar.

Contoh: - Total artikel. - Total waktu membaca. - Topik favorit. -
Aktivitas mingguan. - Progress pembelajaran.

8. External Services

Layanan pihak ketiga: - LLM API (OpenAI, Gemini, Anthropic, Groq,
OpenRouter). - Embedding API. - OAuth Login.

Alur Data

1.  Pengguna membuka artikel.
2.  Extension membaca DOM.
3.  Readability mengekstrak artikel utama.
4.  Teks dibersihkan.
5.  Backend menerima teks.
6.  Backend membangun prompt.
7.  API LLM memproses permintaan.
8.  Backend menerima JSON.
9.  Hasil disimpan ke database dan vector database.
10. Sidebar menampilkan seluruh hasil AI.

Tech Stack

Frontend - React - TypeScript - Tailwind CSS - Vite - Chrome Extension
Manifest V3

Backend - FastAPI - Python - Uvicorn - SQLAlchemy - Pydantic

Database - PostgreSQL - Redis - Qdrant atau ChromaDB

Artificial Intelligence - LLM API - Embedding API

Authentication - JWT - OAuth Google (opsional)

Deployment - Docker - Nginx - GitHub Actions - VPS atau Railway/Render

Monitoring - Sentry - Prometheus - Grafana

Keunggulan Arsitektur

-   Modular sehingga setiap komponen dapat dikembangkan secara
    independen.
-   Mudah mengganti penyedia API AI tanpa mengubah frontend.
-   Mendukung caching untuk mengurangi biaya API.
-   Mendukung semantic search melalui vector database.
-   Mudah diskalakan karena backend bersifat stateless.
-   Siap ditambah fitur seperti multi-model AI, RAG, recommendation
    engine, dan collaborative knowledge sharing.

Pengembangan Lanjutan

-   AI Research Mode.
-   AI Study Planner.
-   Knowledge Graph interaktif.
-   Daily Learning Summary.
-   Team Workspace.
